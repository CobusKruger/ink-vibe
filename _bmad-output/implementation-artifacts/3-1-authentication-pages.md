## Story

As a besoeker,
I want Afrikaans Registreer / Meld aan / Wagwoord-herstel flows,
so that I can create and access my account entirely in Afrikaans. (FR-1)

## Acceptance Criteria

## Dev Notes

### What this story is (and is NOT)

- **IS:** (1) three **Afrikaans single-column auth surfaces** — Registreer / Meld aan / Wagwoord-herstel — built fresh as `ink-foundation` FSE templates **over WordPress's own auth functions** (assembly-only; no Lovable mock); (2) the **`ink-core` account-creation default** (`ink_writer_tier = brons`, gratis lid / no active lidmaatskap) in a new `Ink\Accounts` module wired through the Kernel seam; (3) **Afrikaans transactional auth email** via the Story-1.12 Notifications capability + Afrikaans WP-core mail filters; (4) **authored AND passing** Pest unit tests for the `ink-core` logic. _[epics.md#Story-3.1; architecture.md line 1081; AD-1; AD-9; project-context.md]_
- **IS NOT:** the reader/writer intent gate (**Story 3.2 — REMOVED**; capture no `ink_writer_intent`); registration lifecycle / onboarding / first-action prompt (**Story 3.3** — "follow a skrywer or save a bydrae to leeslys", skippable, graceful on a thin catalogue); anti-spam (3.4, R6); social-login buttons (3.5, R6); the off-by-default "wag op goedkeuring" approval queue (3.6, R6); member/editor **roles + CPT `capability_type` / taxonomy term-caps** (deferred to 3.3 per Epic-2 retro); the **Lidmaatskap purchase/PayFast** flow (Epic 4); profile/account-management screens (Epic 9 / `my-profiel`). No reimplemented auth; no retired login plugin reactivated.

### ⭐ The decided approach (key deliverable) — WP-native auth + theme templates + `ink-core` default

The architecture **decides** this for us — do not re-litigate it:

> **architecture.md line 1081 — Epic→Location:** `3 Accounts & auth | WP-native auth · ink-foundation/templates (login/register) · ink-core (first-action prompt)`
> **ux EXPERIENCE.md line 58:** "Registreer / Meld aan / Wagwoord-herstel — single-column forms, **built fresh in WP**".
> **page-map.csv line 17:** auth → `login|register|forgot-password`, data_source **`wp auth`**, status **`assembly-only`**.

So the split is unambiguous:
- **WordPress** owns the auth *mechanism* (credential storage, session, password hashing, lost-password token) — use it, never rebuild it (and never reactivate Loginizer / Invite Anyone / any retired login add-on).
- **`ink-foundation`** owns the *presentation* — single-column FSE templates/patterns that render the WP auth forms/links in Afrikaans. No business logic.
- **`ink-core`** owns the *business rule* — the new `Ink\Accounts` module sets the account default (Brons + gratis lid) on `user_register` and wires Afrikaans transactional email. (The "first-action prompt" the line mentions is Story 3.3, NOT this story.)

### ⭐ The default mechanism (Brons + gratis lid)

- **Tier:** new accounts get `ink_writer_tier = brons`. Story 2.3 already registers this meta with a **registered default** of `Tier::Brons->value`, so an *unset* value already *reads* as `brons`. This story **materialises** it at `user_register` (`update_user_meta`) so the value is queryable/explicit from account creation and survives a later change to the registered default. Source the key from `Ink\Content\UserMeta::WRITER_TIER` and the value from `Ink\Kernel\Tier::Brons->value` — never a literal. (This mirrors the migration rule "missing/ambiguous tier on import → default `brons` + flag"; here the source is a fresh signup, so plain `brons`, no flag.)
- **Gratis lid:** is the **absence** of an active lidmaatskap — there is nothing to write. Do NOT create a WooCommerce membership, grant entitlement, or touch `Ink\Entitlement`. Submission entitlement is evaluated at the publish moment by Epic 4 / AD-2 (`can_submit()`), independent of tier. A gratis lid may read/react/respond/follow/leeslys (Epic 7/8/9 gate those on `is_user_logged_in()`, not entitlement) but cannot *plaas*.
- **THE conflation rule is the structural guarantee:** `Ink\Accounts` writing tier with zero `Entitlement` reference keeps "Brons writer" and "paid subscriber" separate. _A paid Brons member ≠ a Brons writer with an expired lidmaatskap._
