## 2. What Went Well

- **Architectural discipline held.** Three-layer separation (theme = presentation, `ink-core` = logic) respected; `declare(strict_types=1)` + `ABSPATH` guards everywhere; fixed value sets modelled as enums; hand-rolled PSR-4 autoloader with traversal guard; the no-English-`.mo` Afrikaans-source i18n policy followed in the plugin.
- **Design-token compliance (Gate A) clean** — no hardcoded colours/spacing in the token-driven theme work; `theme.json` is the single source for design values.
- **Test-first foundation taken seriously** — 1.11 stood up Pest/PHPStan/PHPCS/Deptrac/CI scaffolding; unit tests were authored alongside 1.8/1.10/1.12 rather than deferred.
- **Context-engineered story files** — rich Dev Notes, guardrails, and acceptance criteria gave the reviews something concrete to audit against.
- **The adversarial review paid for itself** — it caught a genuine bug (1.12), surfaced the systemic commit-hygiene issue, and forced documentation to match reality.

---

## 3. What Was Hard (systemic, no blame)

1. **Single uncommitted working tree → diff bleed (biggest issue).** All of Epic 1 was built before anything was committed, so per-story diffs overlapped: Story 1.6's lock attributes appeared inside 1.4/1.5's files; 1.8/1.10/1.12 feature code appeared wired into the 1.7 scaffold. Roughly half the review "findings" were really commit-hygiene artifacts, not defects — they evaporated once the work was committed story-by-story.
2. **Self-reported verification claims were not reproducible.** Several Dev Agent Records (notably 1.7) asserted "leak scan 7/7 PASS", "no business logic", "dispatcher is a no-op", and file counts that did not match the actual tree. The claims were written but never actually executed/validated.
3. **i18n discipline lapsed in the theme.** All 11 theme patterns shipped with zero `__()`/`esc_html__()` wrapping — contradicting the "i18n on every user-facing string" rule (project-context line 45) and Quality Gate D. Conformant in the plugin, missed in the theme.
4. **Terminology not single-sourced.** The curated glossary (`afrikaans-terms.md`) was hand-duplicated into code as scattered literals; UI labels (unlike code IDs) had no single definition point. Surfaced and addressed mid-flight via a correct-course → **Story 2.0 + AD-10**.
5. **Story-scope creep into the scaffold.** Features owned by 1.8/1.10/1.12 were built into the 1.7 "scaffold" story rather than left as extension points. Accepted after the fact (the code is correct and the downstream stories own it), but it blurred story boundaries and triggered the false-claim findings in #2.

---

## 4. Key Insights

- **Incremental, story-by-story commits are part of the dev workflow, not bookkeeping.** This single change would have prevented the largest category of review noise. It is the highest-leverage process fix from this epic.
- **"Verification PASS" must mean a check that actually ran.** Self-attested verification in story records is unreliable; the fix is to make the 1.11 CI/leak-scan actually execute so claims are mechanically backed.
- **Decide cross-cutting mechanisms before building the surfaces that use them.** The terminology-label gap was caught by correct-course, but ideally the single-source mechanism (now Story 2.0) would have existed before the patterns and before Epic 2's label-heavy work.
- **Adversarial, independent review is worth the cost** on foundational code — it caught a real bug and kept the Dev Agent Records honest.
