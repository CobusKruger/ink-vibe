## 3. What was harder — challenges & lessons

- **A "deferred" item can already be done — verify before re-fixing.** The carry-forward (and epics.md §14.1) described the `borg` cap divergence as open, but the generic 12.3 `save()` fix already covered every CPT including `borg`, and 3.3 already granted the cap. **Lesson:** treat a deferred-work note as a *hypothesis to verify against current code*, not a fact — the right move was a regression test that proves closure, not a redundant code change. (Memory `epic-13-carryforward` slightly overstated the gap; corrected here.)
- **A pre-existing red gate surfaced at epic start.** The first `composer cs` run showed 2 ERRORS in Epic-13 files (`InkPols\Archive` short ternary, `InkPols\Migration` inline-comment end char) — shipped red at the Epic-13 merge because the 13.x dev cycles cs-checked only the *new-file subset*, not the repo-wide `composer cs`. **Lesson:** run the FULL `composer cs` (not a path-scoped subset) as the gate, so a repo-wide error can't slip through a story whose own files are clean. Fixed both in 14.1 (zero behaviour change) to keep the Epic-14 gate honest. *(New action item below.)*
- **Rotation by `index % count` is only fair/stable for a constant active set.** When a campaign opens/closes mid-day the positional slots remap, so the featured pick can jump within a day — the original docblock overclaimed "stable within a day". **Lesson:** state the precondition (constant membership) explicitly; for the few-sponsors reality the simple modulo is acceptable, but the claim must match the behaviour. (R14 patch.)
- **An inverted window (start > end) fails silently.** A transposed-date typo makes a sponsor never active with no error. **Lesson:** decide the policy deliberately and pin it — chosen fail-closed (never active, no silent date-swap that guesses intent) + a test, converting a surprise into documented behaviour. A save-time validation/notice is the cleaner long-term fix (carried). (R14 edge finding.)
- **Two surfaces, two collapse contracts.** The homepage strip must *fully collapse* (return `''`) with no sponsor, but the recognition section must *always render* the acknowledge+invite copy and only drop the grid. **Lesson:** "graceful when empty" is per-surface, not a single rule — name the contract per block and test both directions non-vacuously.
- **The "build + unit-verified, integration-pending" pattern continues a sixth time.** The active-sponsor `WP_Query`, the featured-image/permalink resolvers, the block registration, the `home_url('/kontak')` CTA target, and the front-page/Oor-INK rendering are unit-tested against mocks (a new minimal `WP_Query` stub was added for the wrapper path) — real-WP behaviour + the Oor INK page assembly are owed to Stories 18.8 / 15.3.
