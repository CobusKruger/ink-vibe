## 3. What didn't — and the code review is the evidence

All nine stories passed their own per-story gates, yet the fresh cross-story adversarial pass still surfaced 6 fixes. The pattern is instructive:

1. **A gate that downgrades an action must re-evaluate ALL its side effects.** 6.8's gate correctly forced `post_status` to draft for a denied plaas, but `attachMedia()` and `linkChallenges()` still ran — so a denied (non-entitled) submission registered `uitdagingsrondte` round entries. The per-story reviews missed it; the **cross-story durability pass** (Epic-5 Action 1) caught it. Patched: challenge linking now runs only after the denial early-return.
2. **Theme i18n is inconsistent.** Two of the highest-prominence strings (the Skryf H1 + intro) shipped as raw literals outside the `ink-foundation` text domain while every other string in the same file was wrapped. The theme patterns have mixed precedent (`auth-login.php` raw H1 vs. `lidmaatskap.php` wrapped). Patched, but the inconsistency is systemic.
3. **Curated copy can be half-applied.** 6.7's success screen used the "Lees en reageer" eyebrow as its H2 and dropped the curated "Gee 'n skrywer vandag 'n hupstoot" prompt block — the AC's "read-and-respond prompts" were reduced to nav links. Patched from the curated source.
4. **Smaller catches:** `MediaAttachment` hooked `init` from within `init` (fragile; fixed to direct registration); `successModel` was spoofable by post id (added an author check); the legacy-routing guardrail's non-vacuity was blind to the theme half (strengthened).

Softer signals:
- **"Delivered but not user-reachable" keeps accumulating** (same as Epics 4/5). The Skryf page needs a WP page at slug `skryf` (editorial, not code); the denial/success flows and — most materially — a path back to a **preserved konsep** need a "My konsepte" surface (Story 9.4). A non-entitled writer's text is preserved as a draft but they currently land on a blank form with no link to it.
- **Mock-only testing compounds on the security-critical path.** `ProseSanitizer`'s actual stripping is delegated to a mocked `wp_kses` — the allowlist *policy* is tested, the *enforcement* only at the 18.8 integration layer. The same mock-only pattern Epic 4/5 flagged.
- **The members-can't-publish capability model is deliberate but non-obvious** — `ink_content` caps are admin/editor-only, so submission is authorised by INK's own rules (logged-in + nonce + entitlement), bypassing WP caps by design. It also means draft creation has no anti-spam/rate guard yet (→ 18.10).

## 9. Key takeaways

1. **The carry-forward mechanism works — use it for every owned, dated debt.** `is_scalar` died because the obligation lived in the auto-loaded memory index, not a retro doc nobody re-reads. Action 4 (anti-spam) is now carried the same way.
2. **A gate must own its side effects, not just its primary write.** The denied-publish path downgraded the status but leaked challenge entries. When an action is denied/downgraded, every side effect of that action is in scope — and only a cross-story pass tends to see it.
3. **Promote proven rules; stop carrying them as candidates.** The cross-story durability pass and non-vacuous guardrails both earned their place this epic — they belong in `project-context.md` now.
4. **The conflation rule is proven from all three sides** (Epic 4 entitlement, Epic 5 Gradering, Epic 6 submission-gate). The core architectural bet has held through its hardest test.
