## Story

As an `ink-core` developer,
I want the unit + integration test harness (Pest/PHPUnit + Brain Monkey/WP_Mock for unit, `wp-env` + WP test library for integration) wired into CI from the start — by standing up the repo-root tooling the architecture reserves (root `composer.json` with the dev-tooling + autoload, `tests/bootstrap.php`, `phpunit.xml`/Pest config, `.wp-env.json`, `phpstan.neon`, `phpcs.xml`, `deptrac.yaml`, `.github/workflows/ci.yml`), reconciling and wiring the two unit tests Stories 1.8 and 1.10 already authored ready-to-run,
so that every P0 `ink-core` rule ships **test-first** (NFR-9 / §14.17): the conflation rule, the entitlement gate, tier promotion, the submission gate and the follow graph all land with their tests against a harness that already exists, rather than the test harness being retrofitted after the rules.
