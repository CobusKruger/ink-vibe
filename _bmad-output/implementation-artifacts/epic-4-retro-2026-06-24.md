## 2. What went well

- **🎯 The headline: tests actually ran — the two-epic-old gap is CLOSED.** Epic 1 action **A2** and Epic 2 critical-path item **#1** ("wire a Pest runner so authored tests actually execute") were the carry-forward of the last two retros. Epic 4 ran the real mocked Pest suite on every story: **120 → 270 passing, zero regressions**. Story 4.2 caught a real idempotency bug red→green (activation re-firing on `active→active`); 4.8's in-place-renewal BLOCKER was caught and proven fixed by test. `project-context.md` was updated to formally **retire** the "no PHP binary, defer to 18.8" precedent. This is the project's biggest process win to date.
- **THE conflation rule held under its hardest test.** Epic 4 is exactly where subscription-vs-tier coupling would be tempting, and it never happened — proven by comment-stripped scans, unit asserts, *and* Deptrac in every one of the 8 stories. `Ink\Entitlement` is now a clean reference seam.
- **Convergent 3-layer reviews caught BLOCKER-class gaps, not nitpicks:** the **in-place-renewal scheduling gap** (4.8 — end-date extends without a status transition, so expiry warnings wouldn't reschedule), the **store-suppression carve-out** (4.6 — checkout / cart-with-lid / PayFast-return endpoints must stay reachable), **price validity** (4.1 — non-positive / non-numeric prices were treated as valid), and the **status-vs-time-authority** refactor (4.3 — grant iff status ∉ REVOKED_STATUSES *and* end-date passes the SAST window).
- **Reuse over duplication, enforced in review.** 4.5 *collapsed* `RenewalPresenter` into a pure pass-through of 4.4's `PlanPresenter` (−5 tests, less surface); 4.8 kept exactly **one** activation-email template rather than duplicating per term. Presentation read-models shape a single cross-module contract; the theme stays presentation-only.
- **Glossary-first Afrikaans discipline never broke.** Zero AI-generated Afrikaans; every string is from `afrikaans-terms.md` or explicitly marked `[NEEDS HUMAN AFRIKAANS]` / `[WAG OP MENSLIKE KOPIE]`.
- **Platform-plugin seaming matured.** Every WooCommerce / WC-Memberships / Action-Scheduler call is behind `function_exists`/`class_exists`/`instanceof` guards with graceful no-op when absent — a now-consistent house pattern (4.1, 4.2, 4.3, 4.6, 4.8).

## 3. What didn't — recurring patterns

1. **A payment epic verified entirely against mocks.** Every WC / WC-Memberships / PayFast / Action-Scheduler call is tested with Brain Monkey, but **the real `purchase → activate → entitle → renew → expiry-warning` round-trip has never executed against real WordPress or the PayFast sandbox.** Deferred to Story 18.8. For a money-handling epic this is the **single largest residual risk**: unit tests prove the *logic*, not that the live WC-Memberships API shapes, hook firing order, and Action-Scheduler timing behave as mocked.
2. **The Afrikaans-copy debt is hardening into a pre-launch gate.** Lifecycle emails (4.8) ship **toggle-OFF** awaiting human copy; status copy (4.7) is partly placeholder. The NFR-1 English-leak detection tooling meant to catch this (Story 17.4 / 18.x) **does not exist yet**, so today the only guard against placeholder leakage is reviewer diligence.
3. **"Delivered but not user-reachable" is accumulating by design.** 4.3's gate has no enforcement point until **6.8**; 4.5's renewal section has no real host until **9.4**; 4.7's status copy isn't rendered until **6.8**. All deliberate and documented — but a growing set of code paths no end-user can exercise yet, which the mock-only testing compounds.
4. **Spec ambiguity surfaced repeatedly at implementation time** (4.1 "what counts as a valid price?"; 4.3 "status is admin-revocation, not the time-authority"; 4.8 "reschedule on end-date change, not just status transition"). Reviews resolved each correctly, but these are semantic contract decisions that ideally land before code.

Smaller signals:
- **`-1 month` warning-anchor overflow** (4.8) accepted as documented imprecision rather than handled — fine at launch, worth a note for the recurring variant (4.10).
- **Activation thank-you fires on *any* `→active` transition** (4.2/4.8), including complimentary/admin grants — copy tone or a paid-only filter is a copy-author decision.

## 9. Key takeaways

1. **The discipline compounded.** Front-loaded context + a clean module seam (`Ink\Entitlement`) + reviews that simplify (collapsing `RenewalPresenter`, one email template) produced a high-velocity, zero-regression epic — and finally closed the test-execution gap that haunted Epics 1–2.
2. **"Tests pass" now means tests ran — but for a payment epic, "tests ran" still doesn't mean "the money path works."** The mock suite is excellent for logic; the un-run PayFast-sandbox round-trip (18.8) is the one thing standing between green CI and launch confidence. It is the top pre-launch item.
3. **Skipping a retro has a cost: lost accountability.** Epic 2's action items (#2, #3) drifted because Epic 3 had no retro to check them. Running the quick Epic 3 retro now recovers that — and is the reason the optional close-step is worth doing.
