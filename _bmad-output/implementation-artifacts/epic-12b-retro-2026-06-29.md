## 2. What went well

- **Value-set discipline held.** `CadenceType` followed the established `Kernel\Tier`/`ResponseType` enum pattern — single-source `values()`/`fromMeta()`/`default()`, no inline `'jaarliks'`/`'maandeliks'` literals; `Content\FieldSets` and `Challenges\Cadence` both consume it.
- **Deptrac caught the right home early.** The story planned `Challenges\CadenceType`, but `Content\FieldSets` needs the enum and `Content` may depend only on `Kernel`. The Kernel placement was resolved during dev (not at review), exactly mirroring why `Tier` is Kernel-owned. No reverse edge shipped.
- **Backward compatibility was airtight.** `fromMeta('')`/absent/junk ⇒ monthly; every legacy round's period derivation is unchanged, proven by the untouched monthly assertions.
- **The adversarial review earned its keep.** Two independent layers (Edge Case Hunter + Acceptance Auditor) converged on the same HIGH that a single self-review missed — see §3.

## 3. What didn't go well — the central lesson

- **🔴 A "reserved capability" behind a vacuous test is not a delivered AC.** The first implementation modelled the annual cadence and made `Cadence` cadence-aware, but **no production code consumed the period** — the winners post titled from the uitdaging title, the banner from the rank label. Setting a round `jaarliks` changed only stored meta. The "pipeline-reuse" test stitched `Cadence::periodLabel → Tiers\Api::winnerLabel` *only inside the test*, so it could pass while the feature did nothing. The story's create-story step even noted "Cadence isn't consumed elsewhere yet" — and that observation should have been a **blocking design question**, not a footnote.
  - **Why it happened:** the story framed the work as "configuration over a cadence-agnostic pipeline / no pipeline-class changes." That framing is half-right (no new *mechanics*) but it let "no consumer" masquerade as "elegantly reused." The pipeline was cadence-agnostic because it ignored the period entirely, not because it reused it across cadences.
  - **The fix (R12B-P1/P2):** `WinnersPost::generate()` now folds the cadence-aware period into the announcement title (`roundPeriod()` → `Cadence::periodLabelFor`), so an annual round publishes "Wenners: 2026 — …" and a monthly one "Wenners: Desember 2026 — …". `periodLabelFor` has a real caller; the vacuous test was replaced by a production-path proof. Bonus: the monthly announcement now carries its period too (it never did), nudging toward the documented "[Maand] wenner" design.
  - **Lesson (promote to practice):** **When a story's deliverable is a capability, the Definition of Done must include a production caller and a test that exercises the production path — never assert a hand-wired composition the system never performs.** This is the project's existing "test the OUTCOME / non-vacuous guardrail" rule applied to *reachability*. Candidate rule for project-context if it recurs.
