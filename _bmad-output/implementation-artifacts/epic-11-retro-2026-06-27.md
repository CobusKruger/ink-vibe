## 2. What went well

- **The Epic-10 Action-4 decision was executed cleanly: a shared archive helper, not a third copy.** `Ink\Kernel\ArchiveRender` now owns the three identical primitives the archives had each copied (`pill`, `pagination`, the defensive `requestInt`/`requestKey`/`requestText` reads). It lives in Kernel (a universal dependency), so **no deptrac edges changed**, and the deferred `pill()` pre-escape hardening from the Epic-10 review was folded in (the URL is now escaped in one place — callers pass raw). Both existing archives were refactored to delegate.
- **A cross-epic refactor was safe because the tests are escape-transparent.** Refactoring Epic-8 (`WorksArchive`) and Epic-10 (`Library\Archive`) production code mid-Epic-11 carried real regression risk — but their unit suites assert on emitted markup with `esc_url`/`esc_html` stubbed as `returnArg`, so moving the escape point into `pill()` is output-identical under test. The suites guarded the refactor and stayed 100% green. **Lesson banked:** a refactor that relocates escaping is provably behaviour-preserving when the suite stubs escaping transparently and asserts output — run the borrowed suites as the regression gate.
- **Pattern reuse again drove velocity.** 11.1's `Hub` is a near-direct port of `Library\Archive`; 11.2's vaardigheid facet mirrors the 10.1 genre filter exactly; 11.4's `RelatedTraining` mirrors `Library\WinnerLinkage`'s relationship-resolver shape (pure `queryArgs` + side-effecting resolver + pure `toHtml`, fail-safe empty). Five built stories, almost no novel infrastructure — the Epic 9/10 "pattern reuse = velocity" finding repeated a third time.
- **The FR-55 "shared-taxonomy surfacing, never manual linking" invariant was built in, and hardened to hold in the pure layer.** `RelatedTraining` matches solely on shared `genre`/`vaardigheid` term ids; a work sharing no term surfaces nothing. The review found that `queryArgs([],[],…)` called *directly* would match everything (safe today via `relatedFor`'s guard) — patched so `queryArgs` itself emits `post__in=[0]` when there are no terms. The invariant now holds at both layers, with a test.
- **Conflation rule + three-layer separation held cleanly.** The `Training` layer was declared `Kernel + Content` only; the Acceptance Auditor confirmed zero `Tiers`/`Entitlement` coupling. Every theme file embeds server blocks + core blocks within locked chrome; the contribution CTA deliberately uses `home_url` + a filter (not `Submission\Api`) to avoid a `Training → Submission` edge.
- **The code review was cheap and convergent — 0 HIGH shipped.** All three layers agreed the core logic was correct; the only actionable items were edge-hardening (a latent direct-call footgun, a misbehaving-filter href, an out-of-range pagination clamp). All small, safe, tested.

## 3. What was harder — challenges & lessons

- **Incrementally evolving one block across stories means editing earlier stories' tests.** `Hub.php` and `HubTest.php` were extended by 11.1 → 11.2 (added the `$facets` param to `toHtml`/`searchHtml`) → 11.3 (re-dressed the shelf, split the empty-state test). Each extension required updating the prior story's test call-sites. This is normal for a block that grows feature-by-feature, but it means "story N is done" can still churn story N−1's tests. **Lesson:** when a sequence of stories incrementally extends one server block, expect and budget for updating the earlier stories' test call-sites; keep method signatures additive where possible (optional params) to minimise it.
- **"Editor's shelf" had to be reconciled to a no-manual-linking design.** 11.3's "Die redakteur se rak" implies editorial curation, which collides with Principle 8 (no per-item manual linking). The resolution — the recency-3 strip *is* the shelf ("Drie stukke om mee te begin." maps exactly onto it) — came straight from `ui-copy-translations.md`, not invented. **Lesson:** when an AC's label suggests a manual-curation feature, check the authored UI copy first; the design usually already resolves it to an automatic proxy.
- **11.5's CTA exposes a real scope seam: the guide-submission pipeline doesn't exist yet.** The "Plaas 'n stuk" button routes to the Epic-6 `/skryf/` flow, but `opleiding_artikel` is not yet a Skryf submittable type — so a reader can *reach* the contribution entry point but cannot yet author a guide end-to-end. This is honestly scoped (AC #5 + a filterable retarget point), but "5/5 done" ≠ contributors-can-publish-guides until the Epic-6 machinery is extended. Tracked, non-blocking.
- **Afrikaans copy-debt now spans 7.x–11.x.** Epic 11 added ~10 authored strings ("Soek in opleiding…", "Die redakteur se rak", "Drie stukke om mee te begin.", "Nog niks op hierdie rak nie.", "Probeer 'n ander soekterm…", "Vee filters uit", "Verwante leerhulpbronne", "Het jy iets om te deel?", the contribution description, "Plaas 'n stuk") — all glossary-consistent authored source already present in `ui-copy-translations.md` (no AI Afrikaans, copy:scan clean), but unratified into `afrikaans-terms.md`. The pre-launch ratification pass is now overdue across five epics.
- **The "build + unit-verified, integration-pending" pattern continues.** The opleiding `tax_query`/`s` archive behaviour, the `RelatedTraining` shared-term resolution against real terms, and the contribution CTA reaching a working guide-submission flow are all unit-tested against mocks — real-WP behaviour is owed to Story 18.8.
