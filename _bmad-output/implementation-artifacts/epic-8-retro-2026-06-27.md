## 2. What went well

- **Server-rendered discovery, no REST (AD-7), paid off in consistency.** Every Ontdek surface is a `WP_Query`-backed block with the pure `queryArgs`/`toHtml`/thin-`render` split inherited from Epic 7's reading templates. The skrywer-card shape defined in 8.3 was reused by 8.5's surfaces and again by Epic 9's ledegids and profile — one card, many entry points.
- **Denormalize-for-sort (AD-7) was applied cleanly.** Trending score, reaction totals, first/last-publish, per-form flags, read counts — all stored on write and ordered by indexed meta, never a live COUNT/JOIN. The sorts are cheap.
- **Diacritic-insensitive search (8.4) was built as a fold-both-sides index**, so `Soren`↔`Sören` self-matches without a collation dependency — a thoughtful, locale-correct design.
- **The conflation rule held even where Tiers is read.** 8.5's "Skrywers in jou Gradering" reads `Tiers\Api` for *display grouping only* — declared as a deptrac Discovery→Tiers edge with a "display, never a gate" comment. This became the precedent for Epic 9's Social→Tiers profile-badge edge.
- **The adversarial review did its job — it caught a feature that was 100% broken.** That is the single most valuable outcome of the epic (see below).

## 3. What was harder — challenges & lessons

- **🔴 The headline lesson: the unit tests asserted the BROKEN behavior, so search shipped completely non-functional and green.** The CRITICAL review finding: `worksQueryArgs`/`skrywersQueryArgs` passed `'%'.$folded.'%'` as the `LIKE` value, but `WP_Meta_Query` *itself* wraps a LIKE value as `'%'.esc_like($v).'%'` — so it double-wrapped and the literal `%` killed every match. **Search returned ZERO results.** And the mocked unit tests were asserting the double-wrapped shape — they *encoded the bug* and passed. This is the most important lesson of the whole project so far:
  - **Mocked unit tests that assert an implementation's SQL/arg shape can pass while the feature is 100% broken.** They verify "the code builds the args I expected," not "the query returns the right rows." When the *expectation itself* is wrong, the test locks the bug in.
  - **Behavior that depends on a third-party contract (here `WP_Meta_Query`'s own LIKE-wrapping) cannot be validated by a mock of that contract.** It needs a real-WP integration test. This is exactly the gap Story 18.8 (wp-env integration layer) is meant to close — Epic 8 is the concrete proof of why it's not optional.
  - **The adversarial code review caught what the suite couldn't** — the Blind Hunter, reading the diff without trusting the tests, spotted the double-wrap. Lesson confirmed: a review layer that distrusts the tests is essential, not redundant.
- **Several review patches were "tests asserted the implementation, not the outcome" in milder form.** The non-writer surfacing ("Skrywers in jou Gradering" returned every Brons member because Brons is the default tier), the epoch-fallback mis-ranking (`post_date_gmt` missing → stored 0 → permanently mis-sorted), and the preview-counting read count were all real-behavior bugs that green unit tests missed. **Lesson:** unit-test the *outcome contract* (what rows/order/count the user sees), and reserve integration tests for the seam where WP's own behavior matters.
- **An AC was satisfied "in code" but had no user-facing affordance.** 8.1's date/archive browse worked by URL only — no UI control until the review added `dateBrowseHtml()`. **Lesson:** "the query supports it" ≠ "the user can do it"; ACs that describe a user action need a rendered affordance, asserted structurally.
- **The denormalize-on-write design has a brownfield blind spot.** Every Discovery denorm + search index is seeded only on a *future* publish/save, so the cloned brownfield DB's already-published content is invisible to the sorts/search until it next saves. Correct for new content, incomplete at launch — owned by the Epic 16 migration backfill. **Lesson:** any denorm-on-write feature carries an implicit "backfill existing data" requirement; name it at design time.
- **The copy-debt lane kept growing.** 8.x sort/surface/filter labels shipped as inline `__()` source literals rather than Terms-registry keys (the genre words *were* registered, so the treatment was inconsistent) — explicitly "consistent with the Epic-7 engagement copy-debt note." This is the same thread that continued into Epic 9; it's now one consolidated pre-launch ratification action.

## 6. Action items

1. **🔴 Make "test the outcome, not the arg-shape" a standing testing rule in `project-context.md`** — promoted from the Epic-8 search CRITICAL. Specifically: (a) for query builders, assert the *result contract* (rows/order/count) where feasible, not just the args struct; (b) NEVER assert a third-party contract's internal shape from a mock (e.g. `WP_Meta_Query` LIKE-wrapping) — that needs an integration test; (c) when a mocked test asserts an arg shape, add a comment naming the integration test that proves the real behavior. ## 8. Key takeaways

1. **🔴 Green mocked tests can certify a completely broken feature.** Test the outcome contract, never a third-party's internal arg shape from a mock — that's what integration tests (18.8) are for. Highest-value lesson to date.
2. **The adversarial review is load-bearing, not ceremonial** — it caught the search CRITICAL the suite missed. Keep the test-distrusting Blind Hunter layer.
3. **Denormalize-on-write implies a backfill obligation** for brownfield data — name it at design time (Epic 16 owns it).
4. **Skipping the Epic 7 retro let the copy-debt thread extend unnamed into Epic 8** — every retro, even `optional`. (All three skipped retros — 7, 8 — are now written.)
