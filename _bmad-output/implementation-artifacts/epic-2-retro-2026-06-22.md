## 2. What went well

- **2.0 as keystone paid off.** Building the glossary-backed terminology registry (`Ink\I18n\Terms`, `ink_term()`, the `ink/term` Block Bindings source, `ink_foundation_term()` theme bridge) *first* meant 2.1/2.2/2.4 all sourced labels from a single source — no inlined Afrikaans, no `__($var)`. This realized Epic 1 action **A4** in full.
- **The registrar pattern became a reusable template.** 2.1's `final class` + `public const` single-source + facade accessor + `register()` loop + private `definitions()` set a house style applied verbatim through 2.5. Velocity accelerated — 2.2 shipped straight to done with zero patches.
- **Story-by-story commits (Epic 1 action A1) fully adopted.** Each story carries its own baseline commit; Epic 1's diff-bleed is gone and reviews ran against clean per-story diffs.
- **Owner-in-the-loop on Afrikaans worked.** Label-casing and invented-plural decisions in 2.0 were escalated to the owner rather than AI-guessed; review caught the one wrong plural (`Skryfwerke` → `Skrywes`).
- **Single-source discipline scaled.** Slugs as class constants, `object_types` sourced from `PostTypes` constants, labels from the registry — enforced by the python3 scan across all stories.

## 3. What didn't — recurring patterns

1. **Tests authored but never executed — again.** Epic 1's #1 action item (**A2**) remains **open**. Every Epic 2 story wrote 'ready-to-run' Pest tests, then deferred execution to Story 18.8 (CI buildout). The python3 scan is reproducible and honest about *structure* but blind to *runtime behavior* — proof being that both recurring bug classes below were caught by human review, not the scan.
2. **Capability stubs that deny everyone** (recurred in 2.1, 2.2, 2.3, 2.4). `MANAGE_TIERS`/`MANAGE_CHALLENGES`/`MANAGE_SPONSORS` are granted to no role, so gated write paths are unwritable by anyone (incl. admins) until Epic 5 role-mapping lands. 2.4 has a REST-vs-meta-box capability divergence.
3. **Non-scalar payload reaching a scalar sanitizer** — the identical `is_scalar()` guard patch was applied three times (2.3 `UserMeta.php`, 2.4 `FieldSets.php`, 2.5 `TermImages.php`). A systemic bug class, not a per-story defect.

Smaller signals:
- **`Terms::has()` fail-soft risk** — a typo'd/removed registry key silently ships a raw machine key to production with `WP_DEBUG` off (deferred in both 2.1 and 2.2).
- **Glossary source-of-truth drift** — `skryfwerk` is the migration catch-all CPT but is absent from `afrikaans-terms.md`.

## 9. Key takeaways

1. **Front-loaded context + a pattern-setting first story (2.0) = high velocity with zero rework.** Repeat this model: decide cross-cutting mechanisms before the surfaces that consume them.
2. **"Verification PASS" still doesn't mean "tests ran."** The structural scan is a good interim, but the A2 gap is now two epics old — closing it (action #1) is the highest-leverage move for Epic 3.
3. **Systemic bug classes deserve systemic fixes.** The `is_scalar` patch applied 3× belongs in a shared template, not in per-story rediscovery.
