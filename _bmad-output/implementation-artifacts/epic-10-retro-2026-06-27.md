## 2. What went well

- **Pattern reuse drove the whole epic.** The `Library\Archive` is a near-direct port of `Discovery\WorksArchive` (pure `queryArgs`/`toHtml` + thin `render`, defensive request reads, `pill()` controls, prev/next pagination, non-vacuous empty-state). `WinnerLinkage` and `AutoUpdate` mirror the Discovery server-block and the Tiers `ink/tier_promoted` event-seam respectively. Three built stories, almost no novel infrastructure — the Epic-9 "pattern reuse = velocity" finding repeated.
- **The conflation rule + three-layer separation held cleanly.** The new `Library` deptrac layer was declared with **only** `Kernel` + `Content` allowed; the Acceptance Auditor confirmed zero `Tiers`/`Entitlement` coupling anywhere in Library (browsing/linkage is open, never gated). All theme files embed server blocks + core blocks with locked chrome — no business logic in the theme.
- **The Epic-9 "pre-flag deptrac edges in create-story" action was applied and it worked.** Every Epic-10 story's Project Structure Notes pre-declared its expected edges (10.1: `Library → Kernel,Content`; 10.5: "none new — covered by 10.1"; 10.6: "`12A → Library` declared when 12A is built"). Result: **zero red `composer deptrac` surprises mid-dev** — the exact friction Epic 9 hit was eliminated. (Action 2 from the Epic-9 retro: ✅ closed.)
- **A genuine cross-cutting modelling improvement: `Content\ChallengeRound`.** 10.5 needed to read the `uitdaging-{id}` round-term slug that 6.6 writes. Rather than duplicate the literal, we single-sourced the convention in `Content` (which owns both the taxonomy and the CPT) and refactored the 6.6 writer to consume it — one definition, two consumers, Submission tests still green. Cleaner than the code it replaced.
- **The R4 stub is a real, tested seam — not a TODO.** 10.6 ships the hook constant + an invocation seam (`Api::notifyWinnerCommitted`) + a wired no-op listener, with `Actions\expectDone`/`expectAdded` tests proving the hook fires (and is fail-safe for a non-positive id). 12A.3 can call it with zero rework; only the listener *body* is deferred. The no-op-body test is non-vacuous (no WP write stubbed → an accidental write would error).
- **The code review converged and was cheap to satisfy.** All three layers independently flagged the same top defect (the search GET-form dropping the active genre); the Auditor additionally caught a literal AC gap (genre badge). Both were small, safe patches. 0 HIGH defects — the cleanest review of the project so far.

## 3. What was harder — challenges & lessons

- **An AC element slipped through dev-story and was caught only at review.** 10.1 AC #5 specified a *genre* badge on cards; the first implementation rendered a redundant *type* badge ("Biblioteekitem") — a literal, written requirement, missed because the port from `WorksArchive` (which shows a *type* badge for its mixed-type listing) was copied a little too faithfully. **Lesson:** when porting a pattern, re-read the new story's ACs against the borrowed code element-by-element; "looks like the precedent" is not "matches the AC." The non-vacuous-test and acceptance-audit layers earned their place again.
- **A GET-form interaction bug that unit tests structurally can't catch.** The search form silently discarded the active genre because an HTML `method=get` form replaces the query string — a browser-behaviour fact, invisible to a PHP unit test asserting the emitted markup. All three review layers caught it by reasoning about runtime behaviour, not markup. **Lesson:** free-text GET forms that must coexist with link-based filters need an explicit hidden-field test (now added) — and this class of "form + URL-state interaction" is owed a real-browser E2E check (→ 18.8). `WorksArchive` sidesteps it by having no free-text form; the Biblioteek's search is the first INK surface to combine a GET form with link filters.
- **Afrikaans copy-debt kept growing — now across 7.x/8.x/9.x/10.x.** Epic 10 added "Uitgelig", "Soek in die biblioteek…", "Soek", "Uit die uitdaging:", and the `Geen %s gevind nie.` reuse — all glossary-*consistent* authored `__()`/`esc_html__()` source in `ink-core` (no AI Afrikaans, nothing leaks, copy:scan clean), but unratified. The pre-launch ratification pass into `afrikaans-terms.md` is now overdue across four epics. (Epic-9 carry-forward item 1 — still owned by the pre-launch copy gate; Epic 10 added to the pile.)
- **The "build + unit-verified, integration-pending" pattern continues.** The archive `tax_query`/`s` query behaviour, the `WinnerLinkage` term→post resolution against real terms, and the `AutoUpdate` hook fired by live 12A ingestion are all unit-tested against mocks — real-WP behaviour is owed to Story 18.8. Honest and flagged, but "6/6 done" ≠ production-ready until 18.8 + the §9.4 organisation analysis.

## 8. Key takeaways

1. **Pre-flagging deptrac edges in create-story works** — adopt permanently; it eliminated the Epic-9 mid-dev friction entirely.
2. **Porting a pattern requires re-reading the new ACs element-by-element** — the genre-badge miss came from copying `WorksArchive`'s type-badge too faithfully.
3. **GET forms + link filters interact through the URL in ways unit tests can't see** — needs an explicit hidden-field test (added) and a browser E2E (18.8).
4. **Single-sourcing a cross-module convention (`ChallengeRound`) at the moment a second consumer appears** is the right call — it left the codebase cleaner than before.
5. **Two formal retros in a row** — the memory-carry-forward mechanism is holding the discipline. Run the Epic 11 retro too.
