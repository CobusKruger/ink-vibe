## 2. What went well

- **The "code + runbook" split made ops-heavy stories honest.** Epic 18 is dominated by infra (Cloudflare, Patchstack, LiteSpeed, host scanning, 2FA plugin) that cannot be code. Every such story shipped the genuine `ink-core` slice (a filter seam, a CLI audit, a pure decision) PLUS a runbook for the running-site parts — never faking admin config as code. Eight runbooks now document the operational truth.
- **Conflation rule held across 7 new modules.** Deptrac confirms zero new Tiers↔Entitlement coupling; new edges are only `Seo→[Kernel,Content]`, `Security→[Kernel]`, `Cache→[Kernel]`. The Acceptance Auditor verified it independently.
- **The Brain-Monkey overridable-seam discipline paid off again.** Every new collaborator put WP/plugin reads behind protected seams (`rankMathActive()`, `currentPostType()`, `isProduction()`, `requesterIp()`, `presentTranslations()`, `userAgent()`/`viewerId()`), so the pure logic unit-tests without WordPress — +98 tests, all mocked, fast.
- **The live leak-gate Layer 2 finally landed** (the Epic-17 carry-forward): a unit-tested `RenderedLeakScanner` detector + a CI/cron crawl CLI + the CI step. The static `copy:scan` ratchet now has its runtime counterpart.
- **Adversarial review caught a real HIGH the gates couldn't.** All gates were green, yet the Blind/Edge-Case hunters both flagged the Cloudflare rate-limit IP bug (REMOTE_ADDR = shared edge IP → global lockout). Gates prove "clean code"; adversarial review proves "correct behaviour."

## 3. What was hard / lessons

- **Two quality-gate gotchas recur and were codified.** (1) `composer stan` needs the Bash sandbox OFF (phpstan's worker binds a TCP port → EPERM). (2) `\WP_CLI::*` calls only satisfy phpstan when they sit AFTER a `class_exists('\WP_CLI')` guard — factoring CLI I/O into a separate method loses the narrowing (hit in 18.2, avoided thereafter). Both saved to project memory ([[ink-stan-sandbox-and-wpcli-phpstan]]).
- **Integration/E2E are authored here but execute in CI — say so plainly.** The unit layer is the only thing that runs in the mocked sandbox; the wp-env/Playwright/crawl layers were authored + PHP-linted but run in CI. 18.8's notes state this explicitly rather than claiming a green run that didn't happen. **Lesson:** for infra-test stories, distinguish "authored + lint-verified" from "executed" in the story record.
- **A naive counter's deferred hardening must actually be built when its epic arrives.** Story 8.3 deferred read-count bot-filtering/dedup to 18.9 in a code comment; 18.10 similarly. Both were honoured (Analytics hardening; RegistrationGuard baseline). **Lesson worth keeping:** a `// deferred to 18.x` comment is a real backlog item — grep for them at epic kickoff.
