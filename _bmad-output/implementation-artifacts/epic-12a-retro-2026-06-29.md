## 2. What went well

- **The whole second launch pillar (editorial-burden automation) landed in one epic, conflation-clean.** Ten new Challenges classes spanning R1 (EntryID + collation) → R2 (ingestion, winners post, moderator feedback, banners, feed) → R3 (auto-promotion wiring), all reading Gradering/placement snapshots only — the Acceptance Auditor confirmed **zero `Ink\Entitlement`** anywhere. THE conflation rule (Gradering ⟂ lidmaatskap) held across the heaviest, most novel epic.
- **Reserved-stub seams let the commit SPINE land before its leaves existed.** 12A.3's `Ingestion` established the full AC-5 commit ORDER (winners-post → moderator-feedback → biblioteek → placements → featured → auto-promotion) with `commitWinnersPost`/`commitModeratorFeedback` as documented reserved no-ops (the 10.6 `AutoUpdate` precedent). 12A.4 and 12A.5 then filled the bodies with zero rework. Building the pipeline first, leaves later, kept each story shippable and tested.
- **"Pasted free text → dekkingsverslag → explicit confirm gate" is the right pattern for a safe irreversible editor action.** No `.docx` parser (owner decision — removes the PhpWord/XXE/zip-bomb surface). The coverage report reconciles the parse against stored EntryIDs, and the confirm gate + hard-block on unknown-id/duplicate mean an imperfect parse can never silently corrupt the commit.
- **Both newly-codified testing rules were exercised for real.** Every story is pure helpers + a thin impure shell over overridable seams (`entriesFor`, `bodyFrame`, `insertComment`, `recordPlacement`, `awardWins`, …) — the Brain-Monkey isolation rule — and every test asserts INK-owned outcomes (the order WE compute, the meta WE write, the markup WE emit), never a mock-of-a-mock. Idempotency tests are non-vacuous (they first prove the write WOULD happen).
- **The adversarial review caught a real data-corruption bug the green gates could not.** `Collation::stripIdentity`'s blunt `str_ireplace` would scrub a short author name ("An", "Roos", "Son") as a SUBSTRING out of the judged work itself. All gates were green; only the Blind/Edge hunters saw it. Same lesson as Epic 18's Cloudflare HIGH — adversarial review earns its keep on the bugs that pass the linters.
- **Idempotency-in-the-write (R12 lesson) held AND was hardened.** `EntryId::assign` first-wins, the per-round `ink_uitdaging_commit_done` marker — and the review added the empty-commit lock guard (a zero-winner commit no longer permanently locks a round) and a defensive per-post dedup so a direct/dirty caller can't double-place or double-award.

## 3. What was harder — challenges & lessons

- **A genuine spec ambiguity surfaced mid-epic: "per Gradering" (12.6) vs "per Gradering AND per category" (12A.3).** 12.6's `Placements::arrange()`/`forRound()` group by Gradering only and collapse to one-per-rank; 12A.3's AC says winners are top-3 per (Gradering × category). Resolved by honoring the literal AC at the GATE (pool = grade×category, editorially correct — poems judged against poems) and flagging the coarser read-view divergence. **Lesson:** when an inserted epic's AC contradicts a shipped story's built model, honor the explicit AC at the new boundary and FLAG the old view for reconciliation — don't silently bend either. The review + retro then own closing the gap.
- **The rank-uniqueness gate was initially keyed on UNTRUSTED input.** 12A.3 first enforced one-per-rank on the *parsed header* grade; the Edge Hunter showed a typo'd/omitted header could let two real-pool rank-1s past the gate. R12A patch 5 re-keyed the gate on the entry's **authoritative stored gradering snapshot**. **Lesson:** an authoritative invariant (readiness flag #1) must be enforced on authoritative data, never on the document the user pasted.
- **The read side of the per-category decision is NOT closed.** The commit gate is now correct, but `Placements::arrange/forRound` (used by the featured slot via `WinnersPost::placedEntries`) still collapses category winners to one-per-Gradering. The DATA is correct (rank stored per entry); only the per-Gradering READ view loses category winners. This needs a deliberate category-aware read-model revision — carried as the #1 follow-up (D1). **Lesson:** a gate fix and a read-model fix are two separate pieces of "honor the per-category decision"; don't conflate them.
- **An irreversible flow's idempotency guard can itself be a footgun.** A zero-winner (or partial-fail) commit would mark the round done and lock it forever (`reeds_gepleeg`), so the real results could never be committed. **Lesson:** the "mark done" must be conditional on having actually written something — idempotency protects against double-writes, but it must not lock in a no-op/partial state.
- **Incrementally growing one module across 7 stories means touching earlier stories' files.** `Ingestion::commitWinnersPost`/`commitModeratorFeedback` filled by 12A.4/12A.5; `Module::register()` grew each story; `FeaturedWinners` gained `orderFeed`. Kept additive (new collaborators, reserved seams, optional params) — the same lesson as Epics 11/12, held.
- **The "build + unit-verified, integration-pending" pattern continues.** The admin screens (collation, ingestion), the `wp_insert_comment` write, the `wp_insert_post` announcement, the `get_comments`/`get_posts` reads, and the auto-promotion against a real `Tiers\Api` are unit-tested against mocks/seams — real-WP behaviour (and the Playwright admin flows) is owed to Story 18.8 (now with a Challenges-adjudication tail).
