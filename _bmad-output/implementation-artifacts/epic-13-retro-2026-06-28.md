## 3. What was harder ‚Äî challenges & lessons

- **`sanitizeDate` validates SHAPE, not validity ‚Äî and two readers disagreed about it.** A `2026-02-30` passes the `FieldSets::sanitizeDate` regex, so `Issue::year()` (digit-position match) grouped it under 2026 while `displayDate()`'s `strtotime` rolled it to March or blanked it ‚Äî the same value, two interpretations. **Lesson:** when a value can be well-shaped but invalid, give it ONE trust point (`normalisedDate()` + `checkdate`) that every reader consumes, so validity is decided once. (R13 P1.)
- **A migration that get-or-creates must count creates vs reconciles, or its report lies.** `run()` did `++$created` for every non-zero id, so a `--force` reconcile was reported as "geskep". **Lesson:** when a write is get-or-create, the summary must distinguish inserts from reconciles ‚Äî the seam should report which it did. *(Cross-module: 12.8 `Challenges\Migration` shares this latent over-count ‚Äî carried as a follow-up to apply the same split when it's next touched.)* (R13 P2.)
- **A naive parser is fragile in three independent ways at once.** `issueDateFromName` matched months as substrings (false positives), returned the first month in *array* order (not string position), and treated any 4-digit run as a year (a volume number misread). **Lesson:** legacy-string parsing needs whole-word matching, position-based selection, and a plausibility range ‚Äî cheap to add, and the review hit all three. (R13 P3.)
- **Migrated data collides on `Y-m-01`, so same-month order must be deterministic.** The migration normalises every issue to the first of its month, so a monthly periodical's same-month issues had identical sort keys and undefined order. **Lesson:** any sort over a key that a migration makes non-unique needs an explicit tiebreak (here, post id). (R13 P4.)
- **`esc_url` is the wrong escaper for a non-HTML context.** `shortcodeFor` used `esc_url`, which entity-encodes `&` ‚Üí `&#038;` and would mangle a query-string PDF URL before the plugin's shortcode parser saw it. **Lesson:** `esc_url` is for HTML output; a URL embedded in a shortcode attribute (or anywhere re-parsed) wants `esc_url_raw`. (R13 P6.)
- **The "build + unit-verified, integration-pending" pattern continues.** The archive `WP_Query`, the attachment-URL resolvers, the `wp_set_script_translations` wiring (handle/timing depend on a plugin not in-repo), and the `wp ink migrate-inkpols` migration are all unit-tested against mocks ‚Äî real-WP behaviour (and the flipbook plugin reconciliation) is owed to Stories 18.8 / 16 / 17.
