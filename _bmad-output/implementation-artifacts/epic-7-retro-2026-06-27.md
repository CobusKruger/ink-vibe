## 2. What went well

- **The "engagement lives in ink-core, not WP comments" non-goal was honored without re-inventing storage.** AD-5a's call — reuse the WP comments *substrate* for Gemeenskapsreaksies (one custom `ink_reaksie` type) while keeping public commenting disabled — gave the team moderation/spam tooling "for free" without violating the "no free-form comments" rule. Clean separation of *substrate* vs *UX*.
- **The "only feedback path" invariant was treated as a module-owned guarantee, not a single-story AC.** Gemeenskapsreaksie being the sole free-form feedback channel held across the reaction, response, and prompt surfaces.
- **Reuse compounded within the epic.** The leeslys store (7.7) and the consolidated line-reaction table (7.3) set the dbDelta + `ON DUPLICATE KEY` + bound-prepare + documented-`phpcs:ignore` pattern that every later custom table (Epic 8 search index, Epic 9 `ink_follows`/`ink_ratings`) mirrored. 7.1's pure `queryArgs`/`toHtml`/thin-`render` reading-template split became the house style for every server-rendered block since.
- **Verb-less counts (7.8) established a durable copy rule.** "342 hartjies", not "342 mense het gereageer" — the `_n()` verb-less pattern that 8.x discovery and 9.x follow/read counts all reuse.
- **Stanza-aware gedig layout (7.2) respected the content's form.** Verbatim, stanza-aware rendering with per-line reaction anchors — a genuinely form-aware reading surface, not a generic prose dump.

## 3. What was harder — challenges & lessons (with hindsight from Epics 8–9)

- **The code review's headline patch was a cross-story durability fix — and it's the pattern that keeps paying off.** Patch 2 added the readable-bydrae write-path gate (`Ink\Engagement\Readable`) across **every** engagement write path (reactions, responses, leeslys), not just the one story that surfaced it — so a crafted REST call can't attach engagement to a Page, the `skryfwerk` bucket, or an attachment. This is exactly the "cross-story durability pass" the Epic-6 retro promoted into `project-context.md`, applied here for the first time post-promotion. **Lesson confirmed:** when a module owns an invariant, test/enforce it across every path that touches it.
- **"Server-render-then-client-flip" shipped its server half but not its client half — and the debt compounded.** The 7.7 `ReadingListToggle` renders correct initial state + `data-ink-*` attributes, but no enqueued JS performs the REST flip (line-reactions.js exists for 7.3; the leeslys/toggle client did not). This **same gap recurred across four Epic-9 surfaces** (Follow/pin/rating toggles + rating form) and is now a consolidated 18.8 item. **Lesson:** the "enqueued client flips it" phrasing in ACs is a recurring half-built seam — 18.8 must own the engagement-client buildout explicitly, or it keeps accreting.
- **Engagement UI-copy debt was opened here and is still unratified.** The "Verwante stukke" heading (7.6), the contextual-prompt copy living in the `ink-core` domain rather than a single-source registry key (7.5), and the `duim op`/`wow→wows` count plurals (7.8) are authored-but-not-glossary-ratified Afrikaans. **This is the same copy-debt lane that 8.x and 9.x kept adding to** — now a standing pre-launch ratification action (Epic 9 retro Action 1). **Lesson:** "authored, copy-debt to ratify" is fine per story but compounds across epics; schedule the ratification pass, don't let it ride to launch.
- **Stored engagement rows are never pruned when their target is unpublished/deleted.** Render-time filters hide orphan leeslys/reaction rows, but the rows persist and accumulate — deferred to an Action Scheduler cleanup (Epic 18 maintenance). A correctness-is-fine-but-grows-unbounded class of debt.
- **A few narrow edge cases were knowingly deferred** rather than over-engineered: the `<br>`-delimited sub-line sharing one reaction anchor (7.2 — the textarea emits `\n`, so `<br>` only arrives via paste), the reaction toggle's missing in-flight disable (7.3 — UNIQUE key prevents corruption, only a wrong final toggle on a fast double-click), and the blank-author edge if a user is deleted mid-write (7.4). All real, all low-impact, all documented. **Lesson:** the team is calibrating deferral well — these were correctly judged not worth blocking on.
