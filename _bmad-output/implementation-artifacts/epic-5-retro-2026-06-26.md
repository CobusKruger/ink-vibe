## 2. What went well

- **The conflation rule held under its second hardest test.** Epic 4 proved `Entitlement ⟂ Tiers`; Epic 5 proved the reverse and 5.6 codified it as runnable guardrails (structural + behavioural) rather than prose. The architectural spine held from both sides.
- **Incremental architecture firsts, each clean.** The `ink_tier_history` table got the subtle activation-ordering call right (include-time `Schema::register()` because the activation hook fires after `plugins_loaded`); the reusable `$wpdb` Mockery pattern and the `ink/tier_promoted` event seam (consumed by 5.10 with no new engine) both landed well.
- **Deliberate build order.** 5.3 (log substrate) before 5.2 (the writer); 5.7 (counter) before 5.8 (engine). Dependencies flowed forward by design, not luck.
- **Reuse / single-source discipline.** 5.9's `progressFor()` reads the same `THRESHOLDS` map 5.8 promotes on (no duplicated 5/15); 5.10 reused the 1.12 Notifications store.
- **Afrikaans discipline never broke.** Email copy (afrikaans-terms.md:213) and the wins-needed subtext (line 74) came straight from the glossary; zero AI-translated strings.

## 3. What didn't — and the code review is the evidence

The build passed its own per-story reviews and all gates, yet a fresh-context, **cross-story** adversarial pass still surfaced 14 fixes. None were conflation violations or security holes, but the pattern is instructive:

1. **Audit-trail fidelity was the recurring theme** (3 findings). `promote()` silently dropped the FR-12 audit row on a failed `record()`; the `ink_tier_history` table was only created at activation (an in-place upgrade would have had no table); and `save()` could mis-attribute a manual change to `actor_id=0` (the *system* sentinel). Each story was clean in isolation — the **durability of the audit guarantee across stories/paths** was the gap. All patched (version-gated `Schema::install()` on `admin_init`; failure seam `ink/tier_promotion_log_failed`; actor-0 guard).
2. **The guardrail story had a guardrail bug.** 5.6's "sole writer" test gated on `WIN_COUNT_META_KEY`, so it only ever matched `promote()` and **missed that `Accounts/Registration` is a legitimate second writer of `ink_writer_tier`** (the brons default at `user_register`). The test passed while its stated invariant was false. Rewritten to an accurate all-module scan allowing `{Tiers, Accounts/Registration}`; spec prose corrected.
3. **Dead/aspirational code.** `GraderingView::colorToken()` returned theme tokens that don't exist and was never called — a latent trap for the 9.4 profile work. Removed.
4. **Admin-form hardening on the one `$_POST` surface (5.2).** Challenge id not re-validated (select-tampering), reason length uncapped, terminal-grade win-count accumulating unbounded — all patched.

Softer signals:
- **"Delivered but not user-reachable" keeps accumulating by design.** 5.4 badge, 5.9 subtext, 5.5 discovery primitives all wait on Epic 9.4 / Epic 8 / Epic 12 to surface (same pattern Epic 4's retro flagged). Deliberate, but the pile grows, and mock-only testing compounds it.
- **5.8's auto-promotion engine has never run against a real challenge result** — it consumes placement records from Story 12A.3, which doesn't exist yet. Correctly scoped to a testable seam, but unexercised end-to-end.

## 9. Key takeaways

1. **A strong build still benefits from a fresh cross-story adversarial pass.** All 14 patches were cross-story / edge issues that per-story review missed; audit-trail durability was the throughline.
2. **A guardrail is only as good as its proven failure mode.** 5.6 shipped a test that could not fail for its own invariant. "Non-vacuous, or it doesn't count" is now the rule.
3. **Small unowned debt compounds.** `is_scalar` survived three retros and regressed in Epic 5. Now owned and dated — not carried a fourth time.
4. **The conflation rule is now proven from both sides** (Epic 4 entitlement, Epic 5 Gradering). The core architectural bet held.
