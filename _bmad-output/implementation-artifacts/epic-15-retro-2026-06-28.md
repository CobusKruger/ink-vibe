## 3. What was harder — challenges & lessons

- **The "docblock self-reference trap" recurred four times.** The non-vacuous guard tests substring-match the WHOLE file, so a docblock that *names* the very token it's guarding against trips the test: `gemeenskap.php`'s comment said "WP_Query", `oor-ink.php`'s said "501(c)(3)", `footer-main.php`'s named the retired plugin ("Ultimate…"), and `kontak.php`/`ContactForm` mentioned the `ink-needs-human-af` marker. **Lesson:** a guard-test sentinel token must not appear in the reviewed file's own prose — describe the forbidden thing without writing the literal (e.g. "no post queries", "US nonprofit legal-status wording", "the retired social-icon plugin", "human-copy-pending marker"). Cheap to fix each time, but it cost a red test run per story; worth pre-empting in create-story by phrasing pattern docblocks around the sentinels. *(New action item.)*
- **An epic can be ordered AFTER its own dependency.** Epic 15.6 needs 12A.4/12A.7, but 12A (inserted as a NEW epic) is entirely backlog — so a P1 story landed before the P-? epic it consumes. The filter-seam decoupling rescued it, but it is a planning smell. **Lesson:** when a story's AC cites a not-yet-built story, design the consumer→producer contract as an explicit seam (a filter/hook) up front and ship the consumer side with a graceful-empty default; flag the producer epic as now-blocking-real-UX. *(12A is now on the critical path for the home winners slot to ever populate.)*
- **A contact form that always says "gestuur" silently loses mail.** The first cut discarded `send()`'s return and always showed success, so a disabled toggle / empty `admin_email` / `wp_mail` failure would tell the visitor a lost message was sent. **Lesson:** for any form whose "handle it" is a side effect (email, write), the visible outcome must reflect whether the side effect actually succeeded — capture the result and message honestly. (R15 MEDIUM patch.)
- **Don't silently descope a spec detail during "simplification".** 15.6's first cut dropped the specified "Lees die volledige storie" read-more link (linked the title instead) and the Completion Notes admitted it — the Acceptance Auditor caught the gap. **Lesson:** a simplification that drops a named AC/Task element is a scope change, not a refactor; either keep it or get the descope agreed. (R15 MEDIUM patch — re-added with the authored copy.)
- **Faithfully copying curated Afrikaans also copies its typos.** "niewinsgericht" (→ niewinsgerig) and "trep" (→ trap) are in `ui-copy-translations.md` and were copied verbatim into `gemeenskap.php`. Per "Afrikaans is source of truth — never AI-retranslate", they were NOT unilaterally corrected in code. **Lesson:** the source-of-truth rule cuts both ways — a typo in the curated copy is fixed at the source by the human author, then re-synced; the dev's job is to flag it, not silently "improve" it. (Deferred to the copy author.)
- **Defensive invariants belong at the consumer too, even when an upstream "owns" them.** `FeaturedWinners::order()` originally trusted the (12A-supplied) payload for rank-uniqueness; a dirty payload would have rendered two "algehele wenners". Added a one-per-rank collapse mirroring `Placements::arrange()`. **Lesson:** the slot that RENDERS the invariant should self-defend even though 12A.3's ingestion is the authoritative dedup — defence in depth for an unbuilt upstream. (R15 patch.)
