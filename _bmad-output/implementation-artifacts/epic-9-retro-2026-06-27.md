## 2. What went well

- **Pattern reuse made velocity high and friction low.** Every store/controller/block mirrored a proven precedent: `FollowStore`/`RatingStore` ← the 7.7 `ReadingListStore` (dbDelta + `ON DUPLICATE KEY` + bound prepares + documented `phpcs:ignore`); the feed/profile/read-count blocks ← the 8.1 `WorksArchive` pure `queryArgs`/`toHtml`/thin-`render` split; the receipt trigger ← the 1.12 form-letter store + 4.8 template registration. Twelve stories, almost no novel infrastructure.
- **The conflation rule + three-layer separation held across all 12 stories.** The `CodeScan` conflation guardrail was extended to all 11 Social follow/pin/rating files; every new deptrac edge (Social→Content, Social→Tiers-**display-only**, Notifications→Social/Content/Entitlement) was declared *deliberately* with a justifying comment, never blanket-allowed. The Acceptance Auditor confirmed: **no Social/Notifications code gates on Tiers/Entitlement.**
- **FR-40 public/private separation was designed up front and double-tested.** 9.4 reserved the `leesgetalle`/`vasgespel` slots; the public `SkrywerProfiel` block deliberately carries no read-count/wins-needed surface, asserted *both* by a rendered-output test *and* a block-source scan. 9.5/9.12 then filled the private slots cleanly. The separation survived the review intact.
- **Graceful-degradation discipline was consistent and tested.** Held-for-moderation (9.6 reviews never surface unapproved — the public read filters `goedgekeur` only, asserted non-vacuously), R7/R8 inert-without-analytics (doubly so for 9.11: no event + empty list), BuddyPress-absent guards (the whole notification + scope layer no-ops cleanly). This matched the R5/R7/R8 sequencing decisions exactly.
- **The one HIGH defect was a real cross-cutting bug, and the review caught it.** The Edge Case Hunter, walking the *shared* 4.8 expiry hook, found that the in-app expiry kennisgewing subscribed to the raw schedule hook and skipped the email's live staleness re-checks — a renewed/revoked member would get a false "expiring" alert. The fix (a post-gate `EVENT_EXPIRY_WARNED` event) is architecturally *cleaner* than the original "one schedule, two outputs" wiring. **This is the cross-story durability pass earning its place again.**

## 3. What was harder — challenges & lessons

- **Brain Monkey test-isolation cost rework twice.** 9.6 needed `ARRAY_A`/`ARRAY_N`/`OBJECT` added to the bootstrap (WordPress not loaded); 9.11 hit the `Notifications\Api` first-wiring-wins (`??=`) static leaking across tests *and* cross-file function-redefinition flakiness on `bp_notifications_add_notification`. **Lesson:** when a facade uses first-wiring-wins statics, reset them via reflection in `beforeEach`; prefer the protected-seam pattern (the 9.1 `buddyPressActive()` seam) over inline `function_exists` for testable guards; keep WP constants in the bootstrap. These are reusable rules, not one-offs (see Action 3).
- **Cross-module deptrac edges were discovered mid-implementation, not anticipated.** The following-feed surfaced an un-allowed Social→Content edge; the fix was promoting `readableTypes()` into `Content\PostTypes` (avoiding a worse Social→Discovery edge) + a ruleset edit. Notifications→Social/Content/Entitlement surfaced the same way in 9.9. **Lesson:** create-story dev-notes should pre-flag the expected deptrac edges so they're declared at design time, not patched after a red `composer deptrac`.
- **Afrikaans copy-debt is accumulating.** `volg`/`volg_tans`/`vasgespel`/`vasgespeld`/`lesing`/`leseroordeel` + several feed/empty-state sentences shipped as glossary-*consistent* authored source flagged "copy-debt to ratify" — the established 8.x lane, no AI Afrikaans, nothing leaks (all via `__()`/`_n()` in ink-core, not theme patterns). But the unratified set is growing across 7.x/8.x/9.x. It needs a ratification pass into `afrikaans-terms.md` before the launch copy gate.
- **A large slice of Epic 9 is "built + unit-verified, integration-pending."** The entire notification layer (9.9/9.11) and the BuddyPress scope (9.1) are guarded seams with **no live BuddyPress in the repo** — correctness is unit-tested, but hook timing, the BP notification-sentence formatter, the mark-all BP reconciliation, and the (now-fixed-but-currently-inert) expiry path are all owed to Story 18.8. This is honest and consistently flagged, but it means "12/12 done" ≠ "production-ready" until 18.8.
- **Server-render-then-client-flip surfaces still ship no client JS.** The Volg / pin / rating toggles + the rating form render correct initial state + `data-ink-*` attributes, but the enqueued flip script does not exist — a **pre-existing** gap (the Epic-7 leeslys toggle is identical), now repeated across four more surfaces. Growing UI debt owed to the 18.8 E2E/client buildout.
